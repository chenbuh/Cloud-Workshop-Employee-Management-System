# 系统主题与皮肤系统 (Theme & Skin System)

## 1. 概述
系统采用了一套动态的主题切换与皮肤定制方案，结合了 Naive UI 的配置能力与原生 CSS 变量的灵活性，实现了深色模式与多种品牌色（皮肤）的无缝切换。

## 2. 核心组件
- **App Store (`@/store/app.ts`)**: 状态中心，负责持久化存储主题偏好，并同步更新 Naive UI 配置与 HTML 全局 CSS 变量。
- **Config Provider (`App.vue`)**: 全局包装组件，根据 Store 状态动态提供 `darkTheme` 和色彩覆盖。
- **Switcher (`MainLayout.vue`)**: 位于顶栏的控制组件，包含主题一键切换与皮肤色盘选择。

## 3. 设计语言
### 3.1 深色模式 (Dark Mode)
- **背景**: 采用深邃的午夜蓝渐变 (`#0f172a` 到 `#1e1b4b`)，而非纯黑，以保持视觉层次感。
- **毛玻璃**: 深色模式下的 `glass-effect` 会调整透明度与模糊度，确保卡片内容清晰。
- **文字**: 使用高对比度的灰白字色 (`#f1f5f9`)。

### 3.2 皮肤系统 (Skin Skins)
目前支持四种精心调配的皮肤配色：
- **靛蓝 (Indigo)**: 默认经典色，稳重且专业。
- **蔷薇 (Rose)**: 充满活力的粉红色系。
- **翡翠 (Emerald)**: 清新的绿色系，适合追求自然感的用户。
- **琥珀 (Amber)**: 温暖的橙黄色系。

## 4. 接入规范
为了使新开发的组件能够自动适配主题切换，请遵循以下开发习惯：
- **颜色使用**: 优先使用 `--primary-color`, `--text-primary`, `--text-secondary` 等系统变量。
- **背景处理**: 使用 `.glass-effect` 类名，它会自动根据深浅色调整背景色与补帧。
- **Naive UI 组件**: 绝大多数 Naive UI 组件已自动适配，无需额外手写 CSS。
